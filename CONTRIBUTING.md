# 贡献指南

感谢您对TRX靓号生成器的关注！我们欢迎所有形式的贡献。

## 如何贡献

### 报告Bug

如果您发现了bug，请：

1. 检查是否已经有相关的issue
2. 创建新的issue，详细描述问题
3. 提供复现步骤和错误信息

### 功能请求

如果您有新功能建议：

1. 检查是否已经有相关的issue
2. 创建新的issue，详细描述功能需求
3. 说明为什么需要这个功能

### 代码贡献

如果您想贡献代码：

1. Fork这个仓库
2. 创建您的特性分支 (`git checkout -b feature/AmazingFeature`)
3. 提交您的更改 (`git commit -m 'Add some AmazingFeature'`)
4. 推送到分支 (`git push origin feature/AmazingFeature`)
5. 打开一个Pull Request

## 开发环境设置

1. 克隆仓库：
```bash
git clone https://github.com/yourusername/trx-vanity-address.git
cd trx-vanity-address
```

2. 安装依赖：
```bash
pip install -r requirements.txt
```

3. 运行测试：
```bash
python test.py
```

## 代码规范

- 使用Python 3.7+
- 遵循PEP 8代码风格
- 添加适当的注释和文档字符串
- 确保所有测试通过

## 提交信息规范

使用清晰的提交信息：

- `feat:` 新功能
- `fix:` bug修复
- `docs:` 文档更新
- `style:` 代码格式调整
- `refactor:` 代码重构
- `test:` 测试相关
- `chore:` 构建过程或辅助工具的变动

## 许可证

通过贡献代码，您同意您的贡献将在MIT许可证下发布。

感谢您的贡献！
